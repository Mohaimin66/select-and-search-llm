# Contributing

Thanks for your interest in contributing. This project is early and moving fast.

## Ways to Contribute
- File issues for bugs or feature requests.
- Improve documentation.
- Implement checkpoints listed in `CHECKPOINTS.md`.

## Development Workflow
1. Fork the repo and create a branch.
2. Make focused changes with clear commit messages.
3. Open a PR with a concise description and screenshots when applicable.

## Code Style
- Prefer clarity over cleverness.
- Keep UI components small and reusable.
- Avoid introducing new dependencies unless necessary.

## Reporting Bugs
- Include macOS version, app version, and steps to reproduce.
